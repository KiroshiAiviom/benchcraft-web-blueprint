# Product specs

Product specs are the **contract** for scope and acceptance criteria.

In this workflow:

- Product specs define **what** and **how we know it’s done** (acceptance criteria).
- ExecPlans define **how we build it** (checkpoint steps).
- Code implements the ExecPlan.

## Where specs live

- `docs/product-specs/` contains feature/epic specs.
- Keep each spec short and focused.

Recommended naming:

- `00-overview.md` (optional) — overall product scope + glossary
- `<slug>.md` — a feature or epic (e.g., `new-user-onboarding.md`)

## Template

- Copy: `docs/product-specs/_TEMPLATE.md`

## How specs connect to plans

- Every ExecPlan should reference at least one product spec.
- Steps should reference specific sections/IDs (avoid re-stating requirements in plans).
