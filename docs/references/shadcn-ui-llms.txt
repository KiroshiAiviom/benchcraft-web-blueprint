SHADCN/UI (LLM REFERENCE)

Purpose:
- Use shadcn/ui as a baseline for common primitives (buttons, inputs, dialogs, etc.)
- Customize via your design tokens (docs/DESIGN_SYSTEM.md), not random tweaks.

Do:
- Keep modifications small and documented.
- Prefer composing primitives over forking large component files.
- Maintain accessible defaults (labels, aria, focus).

Avoid:
- Copying random components from the internet.
- Introducing a second UI library unless explicitly approved.
- Diverging tokens/spacing from the Design System.

If you need a new primitive:
- Propose adding it (and any token changes) in the ExecPlan step sheet before implementing.
