BIOME (LLM REFERENCE)

Biome is used for formatting/linting (when configured in the project).

Typical workflow:
- bun run lint
- fix the underlying issue (don’t “game” lint rules)
- re-run lint/typecheck

Do:
- Prefer the simplest fix that matches the existing code style.
- Keep diffs small and reviewable.

Avoid:
- Blanket rule disables.
- Large formatting churn unrelated to the checkpoint.

If the repo does not have Biome configured yet:
- Mark lint as N/A and explain why (do not introduce Biome unless planned and approved).
