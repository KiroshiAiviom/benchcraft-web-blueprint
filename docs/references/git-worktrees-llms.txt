GIT WORKTREES (LLM REFERENCE)

Use worktrees to keep parallel lanes isolated (builder/tests/cleanup).

Create:
- git worktree add ../wt-<name> -b <branch-name>

List:
- git worktree list

Remove (after merge/apply):
- git worktree remove ../wt-<name>

Rules:
- One worktree = one lane/scope.
- Avoid changing dependencies in multiple worktrees.
- Keep branches named to match the lane and objective.
