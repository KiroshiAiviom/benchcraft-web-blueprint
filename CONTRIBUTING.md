# Contributing

Thanks for taking a look at Benchcraft Web Blueprint.

This repository is an early-stage, docs-first open-source project. The goal is to make disciplined, reviewable, agent-assisted web application work easier to reproduce in public.

## Before opening a PR

- Read [README.md](README.md) for the public project overview.
- Read [AGENTS.md](AGENTS.md) for repo operating rules.
- Read [docs/PLANS.md](docs/PLANS.md) for the checkpoint workflow.
- Keep proposed changes small and reviewable.
- Do not add dependencies without explicit maintainer approval.

## Good first contributions

- Clarify ambiguous docs or broken links.
- Tighten templates or step sheets that remove guesswork.
- Improve public onboarding and examples.
- Add narrowly scoped workflow/reporting improvements.

## How to propose a change

1. Open an issue for larger changes or workflow shifts.
2. For small doc fixes, you can open a direct PR.
3. Explain the problem, the proposed change, and the tradeoffs.
4. Keep claims accurate and avoid marketing language the repo cannot support.

## Pull request expectations

- One focused change per PR when possible.
- Include the reason for the change, not just the diff.
- Update relevant docs if behavior or process changes.
- Avoid scope jumps.

## Dependency policy

Dependency additions, version bumps, and lockfile changes are approval-gated. Propose them first with rationale, alternatives, risks, and exact install commands.

## Code of conduct

By participating in this project, you agree to follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
