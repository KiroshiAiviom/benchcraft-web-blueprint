# Code of Conduct

## Our standard

This project aims to be straightforward, respectful, and useful to contributors with different levels of experience.

Examples of behavior that help:

- Being direct without being insulting
- Explaining technical reasoning clearly
- Giving actionable feedback on docs, workflow, and code
- Respecting review boundaries and stated constraints

Examples of unacceptable behavior:

- Harassment, intimidation, or personal attacks
- Dismissive or hostile communication
- Publishing private or confidential information
- Repeatedly ignoring project rules or review boundaries

## Scope

This code of conduct applies in project discussions, issues, pull requests, and other project spaces.

## Enforcement

The maintainer may remove, edit, or reject contributions and interactions that do not align with this code of conduct.

## Reporting

If you encounter unacceptable behavior, open a private maintainer contact request through GitHub or use another agreed private channel.

## Attribution

This document is a short custom code of conduct derived from common open-source community norms.
